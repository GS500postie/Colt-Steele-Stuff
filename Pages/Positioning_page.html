Anchor positioning

You have an anchor and target. The anchor has to be created. It needs a dashed indent. So we give the anchor element a class name and then create a class and set up the anchor, and use the descriptor 'anchor-name'; (I'm using the class name 'anchor') .anchor {anchor-name: --my-anchor;}

Target. The target element is an element with an absolute position linked to an anchor element matching what's declared on the anchor-name property. This attaches the target element to the anchor element. We set the target with the position-anchor element. .target { position: absolute; position-anchor: --my-anchor;}

Positioning targets. the following two properties are used to set at which side of the anchor element the target is positioned on (position-area) and conditions for hiding the target element when it runs out of room (position-visibility). position-area. Read the aricle about the imaginary grid.OK lets leave that one there because firefox doesn't support anchor positioning yet

Let's get back to standard positioning. Position: static. The element is positioned according to the normal flow of the document. The top, right, bottom,left and z-index properties have no effect. This is the default value.

Position: relative. The element is positioned according to the normal flow of the document, then offset relative to itself based on the values of top, right, bottom, and left. the offset does not affect the position of other elements: thus the space given for the element in the page layout is the same as if the position were static

Absolute. The element is removed from the normal document flow, and no space is created for the element in the page layout. the element is positioned relative to its closest positioned ancestor (if any) or to the initial containing block. Its final position is determined by the values of top, right, bottom, and left.

Fixed. The element is removed from the normal document flow, and no space is created for the element in the page layout. The element is positioned relative to its initial containing block, which is the viewport. its final position is determined by the values of top, right, bottom and left

Sticky. The element is positioned according to the normal flow of the document, and then offset relative to its nearest scrolling ancestor. Essentially the element sticks to the page after a set scrolling motion is activated.

My current problem is that the values of top, right bottom and left are not acting in a predictable way. The fact that the third object is removed from the document flow is correct here. Adding position:relative to the parent element allowed the object to be moved up and down the correct number of pixels but right and left respond weirdly. Something in the parent element or the whole document itself. Basically I worked out that any instance of right or left doesn't offset from the nearest adjacent child (two), but the actual edge of the parent div. So, 'three' here is actually left: 230px which means it's 230px in from the left side. Think I'm getting it! (See the MDN page. It actually explains this. Also the element is 240px in because there is a 10px margin on the left side).
