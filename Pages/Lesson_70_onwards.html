Relative units

  • percentages. These are relative to the PARENT.
  • Ems. 1 em is equal to the computed font-size of the element itself. Or whatever component that has a specified size on the parent. eg. padding and margin. So what I'm ascertaining here is that when you apply ems to something that is not font-size, it calculates the size it should be based on the actual font-size of the element. Confusing but interesting, and make note of that. It does make a lot of sense, especially the way he styled a button that retained its shape despite what size it was. Think about this. It's important for responsive design. JUST REMEMBER THAT EMS ARE RELATIVE TO THE PARENT. If you specify ems to items nested within a parent the ems will adopt the font-size of the IMMEDIATE parent, thus getting larger down the cascade in the parent element.
  • REMS. Relative to the ROOT ELEMENT. This means that if the root element is 20px (or 16px by default I think), the elements will take their sizing from that figure.The rems will multiply off that base figure. Which begs the question, do you set this in your initial reset?
  • There are use-cases for combing rems and ems. For instance having a button that has the font-size based on rems, but the padding should be ems because it is relative to the font size of the immediate parent and doesn't break the design as you resize it.

Opacity and the alpha channel

  • The 'a' in rgba (alpha channel) is different to OPACITY. Rgba affects just the background colour and no other content included in the element. Opacity applies to the CONTENT as well. When you go to the VSCode colour picker there is a transparency slider as well.
